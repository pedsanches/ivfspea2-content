## Alignment decisions (frozen)
- Final submission evidence for IVF/SPEA2 must come only from the dedicated submission rerun.
- Fixed defaults are frozen: `R = 0.10` and `C = 0.11`.
- Submission runs are isolated by run-id range and tag to prevent mixing with historical files.
- RWMOP9 must use robust post-processing for IGD (empirical PF), not direct `GetOptimum` IGD interpretation.
- Manuscript claims and tables must be updated only after matrix completion and artifact regeneration.
- Canonical IVF/SPEA2 code path is inside PlatEMO:
  - `src/matlab/lib/PlatEMO/Algorithms/Multi-objective optimization/IVF-SPEA2`
- Do not add `src/matlab/ivf_spea2` for canonical reruns (legacy mirror; can shadow classes).

## Fixed defaults (frozen)
- Algorithm: `IVFSPEA2`
- IVF parameters: `R = 0.10`, `C = 0.11` (10 mothers + 1 father)
- AR mode parameters: `M=0`, `V=0`, `Cycles=3`, `S=1`, `N_Offspring=1`, `EARN=0`, `N_Obj_Limit=0`

## Scope
- Synthetic benchmarks: 51 instances (ZDT/DTLZ/WFG/MaF)
- Engineering benchmark: RWMOP9
- Runs per configuration: `060`
- Submission run range (isolated): `2001..2060`
- Submission tag: `SUB20260218`
- FE budget per run: `100000`
- Saved snapshots: `10`
- Metrics collected in run: `IGD`, `HV`
- Metrics policy for new experiments: any new experimental campaign (beyond this submission rerun) must, at minimum, compute and persist both IGD and HV for every configuration/problem it evaluates; additional metrics are optional but IGD and HV are mandatory.

## What is considered canonical evidence
- Data files in run-id range `2001..2060` with tag `SUB20260218`.
- Tagged status and manifest files in `results/`.
- Regenerated benchmark tables/figures after matrix completion.
- Robust RWMOP9 outputs generated by `experiments/process_rwmop9.m`.

## What is explicitly NOT allowed
- Mixing submission analysis with legacy runs (`1..100`, `1..60`, `1..30`, etc.).
- Updating manuscript claims from partial rerun outputs.
- Reporting RWMOP9 IGD conclusions without robust empirical-PF processing.
